# Документация API для AccountingScholarships

Этот документ содержит описание всех доступных API-эндпоинтов приложения.

**Базовый URL:** `http://localhost:<port>/api/v1` или `https://localhost:<port>/api/v1`

> **Важно:** Все эндпоинты, за исключением `AuthController`, требуют прохождения авторизации. Не забудьте передать заголовок: \`Authorization: Bearer <Ваш_JWT_токен>\`

---

## 🔐 Auth (Авторизация)

### 1. Авторизация пользователя
- **URL**: `/api/v1/Auth/login`
- **Метод**: `POST`
- **Требует авторизации**: Нет
- **Тело запроса**: `LoginRequest` (логин и пароль)
- **Успешный ответ (200 OK)**: Возвращает JWT токен.
- **Ошибка (401 Unauthorized)**: Неверное имя пользователя или пароль.

### 2. Регистрация пользователя
- **URL**: `/api/v1/Auth/register`
- **Метод**: `POST`
- **Требует авторизации**: Нет
- **Тело запроса**: `RegisterRequest` (данные для регистрации)
- **Успешный ответ (201 Created)**: Возвращает данные созданного пользователя.

## 👥 Students (Студенты в ССО)

### 1. Получить список всех студентов
- **URL**: `/api/v1/Students`
- **Метод**: `GET`
- **Успешный ответ (200 OK)**: Массив всех студентов из локальной БД (ССО).

### 2. Получить студента по ID
- **URL**: `/api/v1/Students/{id}`
- **Метод**: `GET`
- **Параметры пути**: `id` (int) - Уникальный идентификатор студента.
- **Успешный ответ (200 OK)**: Данные студента.
- **Ошибка (404 Not Found)**: Студент не найден.

### 3. Создать нового студента
- **URL**: `/api/v1/Students`
- **Метод**: `POST`
- **Тело запроса**: `CreateStudentRequest` (данные студента).
- **Успешный ответ (201 Created)**: Данные созданного студента.

### 4. Обновить данные студента
- **URL**: `/api/v1/Students/{id}`
- **Метод**: `PUT`
- **Параметры пути**: `id` (int) - ID студента.
- **Тело запроса**: `UpdateStudentRequest` (новые данные).
- **Успешный ответ (200 OK)**: Обновленные данные студента.
- **Ошибка (404 Not Found)**: Студент не найден.

### 5. Удалить студента
- **URL**: `/api/v1/Students/{id}`
- **Метод**: `DELETE`
- **Параметры пути**: `id` (int) - ID студента.
- **Успешный ответ (204 No Content)**: Успешное удаление.
- **Ошибка (404 Not Found)**: Студент не найден.

## 🏦 Epvo (Синхронизация с ЕПВО / Посредник)

### 1. Получить всех студентов из ЕПВО
- **URL**: `/api/v1/Epvo/students`
- **Метод**: `GET`
- **Успешный ответ (200 OK)**: Массив студентов из базы ЕПВО.

### 2. Синхронизировать всех студентов в ЕПВО
- **URL**: `/api/v1/Epvo/sync-to-epvo`
- **Метод**: `POST`
- **Успешный ответ (200 OK)**: Количество синхронизированных студентов.

### 3. Сравнить данные студентов (ССО vs ЕПВО)
- **URL**: `/api/v1/Epvo/compare`
- **Метод**: `GET`
- **Успешный ответ (200 OK)**: Результаты сравнения двух баз для выявления расхождений.

### 4. Синхронизировать конкретного студента по ИИН
- **URL**: `/api/v1/Epvo/sync-student/{iin}`
- **Метод**: `POST`
- **Параметры пути**: `iin` (string) - ИИН студента.
- **Успешный ответ (200 OK)**: Успешная синхронизация.
- **Ошибка (404 Not Found)**: Студент с таким ИИН не найден.

### 5. Пакетная синхронизация (по списку ИИН)
- **URL**: `/api/v1/Epvo/sync-batch`
- **Метод**: `POST`
- **Тело запроса**: `SyncBatchRequest` (массив строк ИИН: `["ИИН1", "ИИН2"]`).
- **Успешный ответ (200 OK)**: Количество синхронизированных студентов.

### 6. Синхронизировать измененных студентов
- **URL**: `/api/v1/Epvo/sync-changed`
- **Метод**: `POST`
- **Успешный ответ (200 OK)**: Количество синхронизированных студентов, у которых были изменения.

### 7. Обновить расчетный счет (IBAN) студента
- **URL**: `/api/v1/Epvo/students/{iin}/iban`
- **Метод**: `PATCH`
- **Параметры пути**: `iin` (string) - ИИН студента.
- **Тело запроса**: `UpdateIbanRequest` (поле `newIban`).
- **Успешный ответ (200 OK)**: Сообщение об успешном обновлении.
- **Ошибка (404 Not Found)**: Студент не найден.
